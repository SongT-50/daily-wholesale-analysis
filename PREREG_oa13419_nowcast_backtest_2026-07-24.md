# PREREG — 가락 경매전 반입(OA-13419) → 대전 익일 낙찰가 예측력 백테스트

> 작성: 2026-07-24 WHOLESALE-T3 (데이터 owner). PIPE #6482 지시 = 예측청구(특허) 되살리기 = **우리에게 유리한 결론** → G-E 엄격. 이 문서는 **결과를 보기 전에 커밋**한다. 커밋 후 해시를 RESULT에 기록한다.
> 정합: `verification-principles.md` §사전등록·§양성 대조군 / `data-to-claim-verification.md` Gate1~3 / measure-preflight.
> ⚠️ **측정 전 청구·판매 금지.** 이 백테스트 통과는 "특허 착수 근거"일 뿐, 그 자체가 상업 표기·청구가 아니다. 통과분도 PIPE G-E → CO 적대검증 → 태은이 판정.

## 1. 질문 / 가설
정산공급(경매後)이 익일 예측 신호가 아님은 이미 H0로 확정(2.5년·8.5년 v2, +1.56/+0.92pp). **본 측정은 새 신호 = 가락 경매전 반입(OA-13419, 실시간 선행)이 다른가**를 *같은 잣대*로 판정.

- **H0 (귀무)**: 가락 경매전 반입[품목, t]은 대전 자기회귀 AR(1) 대비 익일 방향예측을 **의미있게 개선하지 못한다** (+5pp 미달).
- **H1 (대립)**: 반입[품목, t]이 AR(1) baseline 대비 익일 방향예측 적중률을 **≥ +5pp** 개선한다.

## 2. 데이터 (범위 고정 — 결과 보기 전 확정)
- **신호(예측변수)**: OA-13419 `GarakAuctionBefore` 일별 `TOT`[품목, t] = 가락 경매전 반입량(톤). 원천 = 서울 열린데이터광장, 무료·공공누리1유형.
- **타깃**: 대전 낙찰가 `avg_price`[품목, t+1] (base = `daily-wholesale-analysis/supply_price_panel.csv`, market=daejeon).
- **기간**: 두 소스 겹침 = **2023-05-16 ~ 2026-07-23** (OA-13419 최소일 실측). 고정.
- **품목**: 대전 패널 ∩ 가락 GUBUN 정확일치 **18종** (감귤·감자·깻잎·당근·대파·딸기·마늘·무·배추·사과·상추·시금치·양배추·양파·오이·토마토·파프리카·포도). 크로스워크 = 정확명 1:1(스캔 확인, 부류 수산은 필터).

## 3. 모델 & 지표
- **Baseline(귀무모델)** = 대전 AR(1): price[t+1] 방향(상승/하락)을 price[t]·price[t−1]에서만 예측. (H0 측정과 동일 null.)
- **Test 모델** = Baseline + 가락 반입[t] 신호 (반입 수준·전일대비 변화·surge더미). 익일 방향 예측.
- **지표** = **익일 방향예측 적중률(directional hit-rate) 개선폭(pp)**, surge/drop 이벤트 기준(H0 v2와 동일 잣대). 품목별 산출 후 가중 종합.
- **게이트** = Test가 Baseline 대비 **적중률 +5.0pp 이상** (H0 익일예측 게이트와 동일). 미달 = H0 유지(반입도 안 됨).
- **검정** = paired(같은 날짜·품목 쌍에서 두 모델 비교), expanding-window(lookahead 제거), 품목별 최소 관측일 ≥ 60.

## 4. 양성/음성 대조군 (★실패 가능 경로를 *먼저* 박는다)
> §양성대조군: "실패할 수 없는 대조군의 통과는 정보량 0." 아래 둘을 real-signal과 **같은 하네스**로 돌려, 하네스가 (a)진짜 신호를 잡고 (b)가짜를 거른다를 증명한 뒤에야 real-signal 결과를 신뢰한다.

- **PC+ (하네스 생존 = 반드시 통과해야)**: 신호 자리에 **실제 익일 가격변화(누설 타깃)**를 넣음 → 게이트를 크게 초과해야 정상. **실패 경로**: 하네스가 망가졌으면(항상 baseline만 반환) PC+가 +5pp를 못 넘음 → 측정 폐기. = PC+는 실패 가능(정보량>0).
- **NC (노이즈 = 반드시 실패해야)**: 가락 반입의 **TODATE를 무작위 순열**(t→t+1 정렬 파괴) → 게이트 **미달**이어야 정상. **NC가 +5pp를 넘으면** = 하네스가 노이즈에서 허구 구조를 잡는 것 → real-signal 결과 무효.
- **판정 조건**: real-signal 결과는 **PC+ 통과 AND NC 미달**일 때만 유효로 채택. (둘 중 하나 어긋나면 하네스 결함 = 재설계.)

## 5. 제외·가드
- 수산 부류(선어·연체·패류·갑각·젓갈 등) 제외 = 청과 18품목만.
- 휴장일(반입 0·거래 0)·결측일 = 쌍에서 제외(전방채움 금지 = 허위 지속성 방지).
- 집계레벨 검증: 가락 GUBUN이 대전 품목보다 세분/집계 다르면(예 무 vs 열무·알타리) 매핑노트에 기록, 불일치 품목은 민감도분석에서 별도표기.
- 단위 불문(반입=회귀 예측변수, 스케일 무관). 표준화 후 투입.
- **재현성**: 측정 스크립트·이 PREREG·결과 JSON을 커밋. 결과는 RESULT에 코드출력 1:1 인용(STATE 압축표기 금지).

## 6. 결정 규칙
- **게이트 통과(PC+통과·NC미달 하에)**: "가락 경매전 반입이 대전 익일예측에 증분력 있음" = 예측청구 되살리기 **후보 근거** → PIPE G-E → CO 적대검증 → 태은이. (여기서도 청구·판매 표기 금지.)
- **게이트 미달**: H0가 반입 신호에도 유지 = 나우캐스트 프레이밍 하향, 특허는 '데이터 정합성 검증'축으로 전환 검토.
- **대조군 어긋남**: 결과 무효, 하네스 재설계(결론 보고 안 함).

### (c) ★계절 baseline 격상 (원 §3 baseline 대체)
- **탈계절화**: 가락 반입 TOT·대전 avg_price **각 시계열의 자기 월 climatology**로 z-score. z = (x − μ[품목,월]) / σ[품목,월]. μ/σ = 품목×월 평균·표준편차, **각 시계열에서 직접 산출**(반입은 반입 own, 가격은 가격 own).
- 🔴 **expanding-window(train-only) 의무 (CS #6509 catch)**: μ/σ는 **각 시점 t까지의 과거 데이터로만** 산출(각 예측 시점에서 미래 미포함). full-sample 월mean/std 금지 = 미래 계절구조 누수(seasonal-z lookahead, 익일예측 v2 CO MAJOR2와 동형). AR(1) baseline·탈계절 climatology **둘 다 expanding**이어야 반쪽 아님. 최소 warmup(품목별 첫 1~2년)은 산출 불가구간이라 예측대상서 제외.
  - ⚠️ CS seasonal_baseline.json은 **공급(물량) 단위**라 가격 탈계절엔 부적합 → 두 시계열 각자 climatology 자체 산출(방법론 선택, CS #6506에 공유).
- **Baseline(귀무)** = 탈계절 대전가격 z[t+1] 방향을 z[t]·z[t−1]에서 예측(AR on deseasonalized price).
- **Test** = baseline + 탈계절 가락 반입 z[t] (수준 + Δ).
- **게이트** = 방향적중률 **+5pp** (원 §3 불변).
- ⚠️ **한계(정직)**: 월 해상도 climatology = gross 계절confound(겨울↓반입·↑가격)는 제거하나 **월내 미세구조 잔존**. AR항이 단기 지속성 흡수. 게이트가 marginal(경계)이면 **옵션B(day-of-year climatology 신규빌드)를 민감도분석**으로(사전 명시).
