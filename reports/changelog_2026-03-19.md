# 도매시장 분석 시스템 변경 이력
**2026-03-19 적용**

---

## 1. 정산 API 전환 (katRealTime2 → katSale)

- **이전**: katRealTime2 실시간 경매현황 API (경매 시간에만 데이터 존재, 이후 0건)
- **변경**: katSale 정산정보 API (일 정산 완료 후 완전한 데이터)
- **효과**: 누락 없는 전국 도매시장 정산 데이터 수집

## 2. 공판장 데이터 포함 (23개 → 38개 법인)

- **이전**: 청과법인 23개사만 수집 (공판장 누락)
- **변경**: 농협/원협 공판장 15개소 포함, 총 38개 법인 수집
- **대상 공판장**: 농협가락(공), 농협강서(공), 인천원협남촌(공), 인천원협삼산(공), 농협대전(공), 대전원협노은(공), 농협북대구(공), 대구경북원협(공), 농협부산(공), 광주원협(공), 농협광주(공), 전주원협(공), 농협창원(공)

## 3. 공판장 정산 지연 자동 보정 (Backfill)

- **문제**: 공판장(농협/원협)은 청과법인보다 정산 업로드가 1~2일 늦음
- **해결**: 매일 수집 시 D-1, D-2 데이터를 자동 재수집
  - 기존 파일의 건수와 비교하여 신규 데이터가 더 많을 때만 업데이트
  - 업데이트된 날짜는 대시보드 요약도 자동 재생성
- **파이프라인**: `[0] 공판장 보정 → [1] 수집 → [2] 출하예약 → [3] AI분석 → [4] 전일비교 → [5] 대시보드 재생성`

## 4. 물량(톤) 표시 오류 수정

- **이전**: 건수와 물량이 동일하게 표시됨 (quantity 필드 사용, 항상 1)
- **원인**: 정산 API의 `quantity`는 항상 1이고, 실제 물량은 `unit_weight`(kg)에 있음
- **수정**: `kg = unit_weight × quantity` 로 실제 물량 계산
- **영향 파일**: `preprocess.py` (대시보드 요약), `index.html` (톤 단위 표시)

## 5. DJC 경영 리포트 — 대전 비교 데이터 자동 선택

- **문제**: 당일 데이터에 공판장이 없으면 대전 4개 법인 비교가 불완전
- **해결**: `_find_complete_date()` 함수로 공판장 포함된 가장 최근 날짜 자동 탐색 (D-0 ~ D-3)
- **리포트 명시**: 비교 날짜가 다를 경우 "⚠️ 공판장 정산 지연으로 {날짜} 데이터 기준 비교" 표시
- **데이터 안내 섹션 추가**: 왜 비교 데이터가 다른 날짜인지 설명

## 6. 대시보드 자동 날짜 탐색

- **이전**: 날짜 하드코딩 (수동 변경 필요)
- **변경**: 오늘 날짜부터 최대 7일 전까지 자동 탐색하여 가장 최근 데이터 표시
